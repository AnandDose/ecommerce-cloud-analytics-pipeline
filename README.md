# End-to-End E-Commerce Cloud Analytics & Statistical Modeling Pipeline

A full-stack data engineering and analytics infrastructure designed to track customer attrition threat surfaces and product price elasticity of demand. This project orchestrates a programmatic ELT pipeline using an AWS data lake landing zone, a multi-layered Snowflake cloud data warehouse, and a containerized Python statistical engine, culminating in an executive-ready star-schema semantic dashboard inside Power BI.

---

## 🏗️ Core Architecture & Tech Stack

The architecture follows a modern cloud data stack design, transitioning decoupled datasets from landing zones to predictive business intelligence assets:

* **Cloud Storage (Data Lake):** AWS S3 (Staging landing zone for raw transactional mutations)
* **Cloud Data Warehouse (DWH):** Snowflake (Multi-layered relational storage splitting `RAW` ingestion and `ANALYTICS` presentation layers)
* **Orchestration & Analytics:** Python (statsmodels, pandas, snowflake-connector)
* **BI Visualization Layer:** Power BI Desktop (Star-schema semantic modeling & native DAX computation)

## 📊 Statistical Methodologies & Data Modeling

### 1. Price Elasticity of Demand (Log-Log Linear Regression)
To discover exact consumer price-sensitivity thresholds across retail product categories, the Python analytical engine automatically applies a constant elasticity model:

$$\log(\text{Quantity}) = \beta_0 + \beta_1 \cdot \log(\text{Price Paid}) + \epsilon$$

* **Interpretation:** The regression coefficient ($\beta_1$) represents the price elasticity metric. Categories displaying $\beta_1 < -1$ are flagged as **Elastic** (highly responsive to promotional adjustments), while values between $0$ and $-1$ register as **Inelastic** (price changes do not materially decay transaction volumes).

### 2. Behavioral Customer Churn Risk (Quantile-Based RFM Segmentation)
Rather than relying on static timelines, the pipeline monitors transactional telemetry across three customer indices:
* **Recency ($R$):** Operational days elapsed since an individual user's latest transaction timestamp.
* **Frequency ($F$):** Total historical volume of execution transactions logged.
* **Monetary Value ($M$):** Aggregate net capital expenditure across the entire user lifespan.

The framework scores consumer actions using statistical quantiles (percentile splits from 1 to 4) to combine structural risk parameters. Scores $\ge 9$ are actively isolated into the **Critical Churn Risk** cohort to signal immediate marketing intervention vector needs.
